# 进程与线程


**进程是资源分配的最小单位，线程是 CPU 调度的最小单位。**

## 四个维度对比

| 维度          | 进程                                 | 线程                            |
| ----------- | ---------------------------------- | ----------------------------- |
| **资源**      | 独立内存空间、文件句柄、信号，资源隔离强               | 共享进程的堆、代码段、文件描述符，资源开销小        |
| **创建与切换开销** | 创建需分配 PCB 和虚拟地址空间，切换需换页表刷新 TLB，开销大 | 创建轻量级直接复用进程资源，切换仅保存寄存器栈指针，开销小 |
| **通信**      | IPC（管道、共享内存、信号、Socket），需内核介入，效率低   | 直接读写共享变量，无需内核介入，效率高但需加锁防竞态    |
| **稳定性**     | 一个进程崩溃不影响其他进程，隔离性好                 | 一条线程崩溃整个进程退出，一个挂全挂            |
