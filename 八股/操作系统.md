# 进程与线程


**进程是资源分配的最小单位，线程是 CPU 调度的最小单位。**

## 两者区别

| 维度   | 进程                          | 线程               |
| ---- | --------------------------- | ---------------- |
| 资源拥有 | 独立内存空间、文件句柄、信号              | 共享进程的堆、代码段、文件描述符 |
| 切换开销 | 大（切换页表、刷新 TLB）              | 小（仅保存寄存器、栈指针）    |
| 创建销毁 | 慢（系统调用分配 PCB、虚拟地址空间）        | 快（轻量级，共享已有资源）    |
| 通信方式 | IPC（管道、共享内存、信号、Socket）      | 直接读写共享变量         |
| 隔离性  | 强，一个进程崩溃不影响其他               | 弱，一条线程崩溃整个进程退出   |
| 地址空间 | 独立 4G 虚拟地址空间（内核 1G + 用户 3G） | 共享所属进程的地址空间      |
| 调度   | 进程是资源的容器，本身不参与调度            | 线程是 CPU 调度的基本单位  |
| 关系   | 一个进程至少包含一个主线程               | 线程必须依附于进程存在      |
