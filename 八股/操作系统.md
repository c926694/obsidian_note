# 进程与线程


**进程是资源分配的最小单位，线程是 CPU 调度的最小单位。**

## 两者区别

| 维度          | 进程                                 | 线程                         |
| ----------- | ---------------------------------- | -------------------------- |
| **资源分配**    | 独立内存空间、文件句柄、信号，资源隔离强               | 共享进程的堆、代码段、文件描述符，资源开销小     |
| **创建与切换开销** | 创建需分配 PCB 和虚拟地址空间，切换需换页表刷新 TLB，开销大 | 创建直接复用进程资源，切换仅保存寄存器栈指针，开销小 |
| **通信**      | IPC（管道、共享内存、信号、Socket），需内核介入，效率低   | 直接读写共享变量，无需内核介入，效率高但需加锁防竞态 |
| **稳定性**     | 一个进程崩溃不影响其他进程，隔离性好                 | 一条线程崩溃整个进程退出，一个挂全挂         |

## 进程通信方式（IPC）
1. **管道（Pipe）**：包括匿名管道（父子进程间）和命名管道（FIFO，任意进程间）。
2. **消息队列（Message Queue）**：进程间通过消息队列传递结构化数据。
3. **共享内存（Shared Memory）**：一块内存区域映射到多个进程的地址空间，效率最高，但需要信号量同步。
4. **信号量（Semaphore）**：用于进程间同步，不传数据。
5. **信号（Signal）**：异步通知机制，如 SIGINT、SIGKILL。
6. **Socket**：可用于同一主机或不同主机的进程通信。
### 追问

**Q: 哪种 IPC 最快？**
共享内存。因为它直接把物理内存映射到进程地址空间，数据在内存里读写，不需要经过内核拷贝。其他方式（管道、消息队列、Socket）都需要内核做一次数据拷贝。

**Q: 共享内存有什么问题？**
多个进程同时写同一块内存会有竞态条件，所以必须配合信号量做同步。

**Q: 管道和消息队列的区别？**
管道是字节流无格式，读出来什么就是什么；消息队列有类型和格式，接收方可以按类型选择性地读。

**Q: Socket 和前面的有什么不同？**
只有 Socket 支持**跨网络**通信，其他都是同一台机器上的 IPC。HTTP、RPC、数据库连接底层都是 Socket。**Socket 和管道的本质**都是内核缓冲区 + 读写两端。**Socket 只是在内核缓冲区外面多套了一层网络协议栈**，所以能跨机器。而管道简单粗暴，连命名都没有（匿名管道只能父子进程用）。**所以说 Socket 就是管道的升级版，能跨网络的管道。**
