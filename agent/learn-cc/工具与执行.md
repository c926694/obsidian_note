# Messages API 响应结构

```
res (Message)  ← client.messages.create() 的返回值
├── id: "msg_01..."
├── type: "message"
├── role: "assistant"
├── model: "claude-sonnet-4-6"
├── stop_reason: "end_turn" | "tool_use" | "refusal" | "max_tokens"
├── stop_sequence: None
├── stop_details: { type, category, explanation }  # 仅 refusal 时有
├── usage: { input_tokens, output_tokens, cache_*, ... }
│
└── content: list[ContentBlock]  ← ★ 真正的内容在这里遍历
    │
    ├── { type: "text",          text: "我来帮你..." }
    ├── { type: "thinking",      thinking: "先分析需求...", signature: "..." }
    ├── { type: "tool_use",      id: "toolu_...",  name: "bash",
    │                            input: {"command": "ls -la"} }
    ├── { type: "tool_use",      id: "toolu_...",  name: "read",
    │                            input: {"path": "file.txt"} }
    └── { type: "text",          text: "完成了。" }
                    ↑
         agent_loop 里遍历的就是这个数组
         for block in response.content:
```

**你追加 results 时的嵌套关系：**

```
messages.append({"role": "user", "content": results})
                              ↑              ↑
                              role           content 是 list[ContentBlock]
                                             里面装的是：
                                             [
                                               { type: "tool_result",
                                                 tool_use_id: "toolu_...",
                                                 content: "命令输出" },
                                               { type: "tool_result", ... },
                                             ]
```

**对应到你的 agent_loop：**

| 代码 | 含义 |
|------|------|
| `response = client.messages.create(...)` | 拿到 `res` (Message) |
| `response.content` | 取出 content 数组 |
| `block.type` | 判断 block 类型：`"tool_use"` / `"text"` / `"thinking"` |
| `block.name` | 工具名：`"bash"` |
| `block.input` | 工具参数：`{"command": "ls"}` |
| `block.input["command"]` | 取出 command 参数传给 `run_bash()` |
| `response.stop_reason` | 判断是否还有更多工具要调 |

# agent-loop
## 主循环
一个工具+一个循环=agent
llm call后根据tool_calls表查看需要调用的工具，然后对工具名进行匹配后获取参数进行工具调用
```py
def agent_loop(messages: list):

    while True:

        response = client.messages.create(

            model=MODEL, system=SYSTEM, messages=messages,

            tools=TOOLS, max_tokens=8000,

        )
        # Append assistant turn

        messages.append({"role": "assistant", "content": response.content})

  

        # If the model didn't call a tool, we're done

        if response.stop_reason != "tool_use":

            return

        # Execute each tool call, collect results

        results = []

        for block in response.content:

            if block.type == "tool_use":

                print(f"\033[33m$ {block.input['command']}\033[0m")

                output = run_bash(block.input["command"])

                print(output[:200])

                results.append({

                    "type": "tool_result",

                    "tool_use_id": block.id,

                    "content": output,

                })

        messages.append({"role": "user", "content": results})
```

## 执行函数
```py

SYSTEM = f"You are a coding agent at {os.getcwd()}. Use bash to solve tasks. Act, don't explain."
  
  
def run_bash(command: str) -> str:

    dangerous = ["rm -rf /", "sudo", "shutdown", "reboot", "> /dev/"]

    if any(d in command for d in dangerous):

        return "Error: Dangerous command blocked"

    try:

        r = subprocess.run(command, shell=True, cwd=os.getcwd(),

                           capture_output=True, text=True, timeout=120)

        out = (r.stdout + r.stderr).strip()

        return out[:50000] if out else "(no output)"

    except subprocess.TimeoutExpired:

        return "Error: Timeout (120s)"

    except (FileNotFoundError, OSError) as e:

        return f"Error: {e}"
```

```py
if __name__ == "__main__":

    print("s01: Agent Loop")

    print("输入问题，回车发送。输入 q 退出。\n")
    history = []
    while True:
        try:

            query = input("\033[36ms01 >> \033[0m")

        except (EOFError, KeyboardInterrupt):

            break

        if query.strip().lower() in ("q", "exit", ""):

            break

        history.append({"role": "user", "content": query})

        agent_loop(history)

        # Print the model's final text response

        response_content = history[-1]["content"]

        if isinstance(response_content, list):

            for block in response_content:

                if getattr(block, "type", None) == "text":

                    print(block.text)
        print()
```
# tool-use
_"加一个工具, 只加一个 handler"_ — 循环不用动, 新工具注册进 dispatch map 就行。
**Harness 层**: 工具分发 — 扩展模型能触达的边界。

## 工具分发映射
```py
TOOL_HANDLERS = {  
    "bash": run_bash, "read_file": run_read, "write_file": run_write,  
    "edit_file": run_edit, "glob": run_glob,  
}
```
工具的使用查表就行
```py
for block in response.content:  
    if block.type == "tool_use":  
        print(f"\033[33m> {block.name}\033[0m")  
        handler = TOOL_HANDLERS.get(block.name)  
        output = handler(**block.input) if handler else f"Unknown: {block.name}"  
        print(str(output)[:200])  
        results.append({"type": "tool_result", "tool_use_id": block.id, "content": output})  
  
messages.append({"role": "user", "content": results})
```
